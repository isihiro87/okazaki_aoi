# 公式LINE 開設・接続の手順

公式アカウントの作成から、本フォルダの実装（あいさつ・登録フォーム・リッチメニュー）に
つなぎ込むまでの手順です。**上から順に**進めれば動きます。所要：初回 1〜2時間。

> 用意するもの：作業用のLINEアカウント（スマホ）、Googleアカウント、PCのブラウザ。
> 最後に必要な差し替え値は **§8 差し替え一覧** にまとめてあります。

---

## 全体像（5つのID/URLを集める作業）

| 集めるもの | 取得元 | 使い道 |
|---|---|---|
| チャネルアクセストークン | LINE Developers（Messaging API） | リッチメニュー登録（curl） |
| LIFF ID | LINE Developers（LIFF） | 登録フォーム連携・メニューの「会員登録」 |
| スプレッドシートID | Googleスプレッドシート | GASの保存先 |
| GAS WebアプリURL | Apps Script デプロイ | 登録フォームの送信先 |
| 公開サイトURL | サイトのホスティング | メニューの「講話スケジュール／会場アクセス」 |

---

## 1. LINE公式アカウントを作成

1. **LINE Official Account Manager**（manager.line.biz）に、LINEアカウントでログイン。
2. 「アカウントを作成」→ 会社/事業者として **「岡崎市葵倫理法人会」** で作成。
3. 作成後、基本情報（アイコン・カバー・説明・所在地）を設定。
   - アイコン：会のロゴ、説明：MS案内、と入れておく。

## 2. Messaging API を有効化（＝LINE Developers と連携）

1. Official Account Manager の **設定 > Messaging API** を開く。
2. 「Messaging APIを利用する」→ **プロバイダー** を作成（例「岡崎市葵倫理法人会」）。
3. これで **LINE Developers Console**（developers.line.biz）に、この公式アカウントに対応した
   **Messaging API チャネル** が作られる。

## 3. チャネルアクセストークンと応答設定

1. **LINE Developers Console** で 2 のチャネルを開く。
2. 「Messaging API設定」タブ → **チャネルアクセストークン（長期）** を発行する。
   - **保存場所**：このリポジトリ／ファイルには**保存しない**。トークンはコンソールでいつでも再表示・再発行できる。
   - 今回使うのは §8 のリッチメニュー登録 curl の一度きり。**その時だけコンソールからコピー**して使い、終えたら破棄でよい。
   - curl を打つ間だけ環境変数に入れる（履歴・ファイルに残さない）：
     ```powershell
     $env:LINE_TOKEN = "ここにトークン"   # PowerShell。ターミナルを閉じれば消える
     ```
   - チームで保管が必要なら**パスワードマネージャー（役職者のみ閲覧）**へ。平文での共有メモ・チャット・スクショは不可（設計書 §5.2）。
   - 将来GASで配信/Webhookを作る際は、コード直書きせず**GASのスクリプトプロパティ**に保存して読み出す。
3. 応答設定（Official Account Manager の **設定 > 応答設定**）：
   - **あいさつメッセージ：オン**（§5で内容を設定）
   - **応答メッセージ**：必要に応じてオン（自動応答）。
   - **Webhook**：今回の最小構成では未使用（出欠・トリガー配信を作る段階でオンにする）。
   - 「チャット」をオンにすると 1to1 対応が可能（設計書 §3.1）。

## 4. LIFF アプリを作成（登録フォーム用）

1. 先に **登録フォーム（`liff-register/`）を公開**しておく（任意の静的ホスティング。公開サイトと同じ場所でよい）。
   - 公開URL：`https://okazaki-aoi.vercel.app/line/liff-register/`（Vercelデプロイ済み）
2. LINE Developers Console の同じチャネル → **LIFF** タブ → 「追加」。
   - サイズ：**Full**
   - エンドポイントURL：上記フォームのURL
   - Scope：`profile`（氏名取得に使用）にチェック
3. 発行された **LIFF ID**（`0000000000-xxxxxxxx` 形式）を控える。
   - フォームを開くURLは `https://liff.line.me/＜LIFF ID＞`。

## 5. あいさつメッセージを設定

1. `あいさつメッセージ.md` の「① 友だち追加時のあいさつ」を、サイトURL等を実値にして用意。
2. Official Account Manager の **あいさつメッセージ** に貼り付けて保存。
3. （任意）「体験参加希望」自動応答（同ファイル④）を応答メッセージに設定。

## 6. 保存先（スプレッドシート＋GAS）を用意

`gas/README.md` の手順どおり：
1. スプレッドシートを作成 → **スプレッドシートID** を控える。
2. Apps Script に `Code.gs` を貼り、`SHEET_ID` を差し替え。
3. ウェブアプリとして公開（実行：自分／アクセス：全員）→ **WebアプリURL** を控える。

## 7. 実装をつなぎ込む（差し替え）

§8の表に従い、控えた値を各ファイルに入れる。
- `liff-register/js/app.js`：`LIFF_ID` と `GAS_ENDPOINT`
- `richmenu/richmenu.json`：`会員登録` の `uri`（`https://liff.line.me/＜LIFF ID＞`）、`講話スケジュール`/`会場アクセス` の `uri`（サイトURL）

## 8. リッチメニューを登録

画像は `richmenu/richmenu-image.jpg`（2500×1686・約0.6MB）が配置済み。
「定義登録 → 画像アップロード → 既定メニュー設定」をAPIで行う。

### トークンを安全に使う実行例（PowerShell）

トークンは**ファイルに保存せず、ターミナルの環境変数に一時的に入れて**使う。
`Read-Host` で貼り付ければコマンド履歴にも残らない。終わったらターミナルを閉じれば変数も消える。

```powershell
$env:LINE_TOKEN = Read-Host "チャネルアクセストークンを貼り付け"   # 入力欄に貼る＝履歴に残らない
cd C:\Users\user\projects\rinri\official_line\line\richmenu

# 1) 定義を登録し、返ってきた richMenuId を取得
$rid = (curl.exe -s -X POST https://api.line.me/v2/bot/richmenu `
  -H "Authorization: Bearer $env:LINE_TOKEN" -H "Content-Type: application/json" `
  -d "@richmenu.json" | ConvertFrom-Json).richMenuId

# 2) 画像（JPEG）をアップロード
curl.exe -X POST "https://api-data.line.me/v2/bot/richmenu/$rid/content" `
  -H "Authorization: Bearer $env:LINE_TOKEN" -H "Content-Type: image/jpeg" `
  --data-binary "@richmenu-image.jpg"

# 3) 全ユーザーの既定メニューに設定
curl.exe -X POST "https://api.line.me/v2/bot/user/all/richmenu/$rid" `
  -H "Authorization: Bearer $env:LINE_TOKEN"
```

> ⚠️ 登録前に `richmenu.json` の「会員登録」URIが **LIFF ID 実値**になっていること（§4・§7）。未設定のままだと会員登録ボタンだけリンク切れになる。
> 画像だけ先に出したい場合は、API登録ではなく **Official Account Manager（GUI）でリッチメニュー作成 → 画像アップロード**でもよい。

> GUIでもAPIでも、保存・共有が必要なら**パスワードマネージャー**へ。`.env` や `.md`/`.txt` での平文保存・チャット共有は不可（§3）。

## 9. 動作確認

- [ ] スマホで公式アカウントを**友だち追加** → あいさつが届く。
- [ ] リッチメニュー4ボタンがそれぞれ正しく開く／送信される。
- [ ] 「会員登録」→ フォームが開き、氏名が自動表示される。
- [ ] テスト登録 → スプレッドシート `members` に行が追加される。
- [ ] 同じ人がもう一度登録 → 行が**上書き**される（増えない）。
- [ ] 「お問い合わせ」送信 → チャット/自動応答で受けられる。

## つまずきやすい点

- **LIFFは「同じMessaging APIチャネル」に追加**すればよい（別途LINEログインチャネルを作らなくても可）。
- フォームを開いて氏名が出ない＝LINE外（PCブラウザ）で開いている。LINEアプリ内＝リッチメニュー経由で開くと連携される。
- 登録が保存されない＝GASの公開設定が「アクセス：全員」になっていない／`GAS_ENDPOINT` の貼り間違い。
- リッチメニュー画像エラー＝サイズが 2500×1686 でない／1MB超過。
- トークン・スプレッドシートはセンシティブ情報（設計書 §5.2）。共有範囲を役職者に限定する。

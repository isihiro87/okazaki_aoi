# GAS（登録フォームの保存先）セットアップ

会員登録フォームの送信先。Googleスプレッドシートに会員情報を保存します。

## 手順

1. **スプレッドシートを作成**
   - 任意の名前で新規作成（例「岡崎市葵_会員データ」）。
   - URL の `/d/` と `/edit` の間が **スプレッドシートID**。控えておく。
   - シートは空でOK（初回送信時に `members` シートとヘッダーを自動作成）。

2. **Apps Script を開く**
   - スプレッドシートのメニュー「拡張機能 > Apps Script」。
   - `Code.gs` の内容を貼り付け、先頭の `SHEET_ID` を上記IDに差し替えて保存。

3. **ウェブアプリとして公開**
   - 右上「デプロイ > 新しいデプロイ」→ 種類「ウェブアプリ」。
   - 説明：任意／次のユーザーとして実行：**自分**／アクセスできるユーザー：**全員**。
   - デプロイ → 表示される **ウェブアプリ URL** を控える（`liff-register/js/app.js` の `GAS_ENDPOINT` に設定）。

4. **動作確認**
   - ウェブアプリ URL をブラウザで開いて `{"ok":true,...}` が返ればOK（`doGet`）。

## 保存される列（members シート ／ 設計書 §2.5 相当）

| 列 | 内容 |
|---|---|
| 登録日時 | 送信日時 |
| LINEユーザーID | LIFFで取得した userId（更新時のキー） |
| LINE表示名 | LINEの表示名 |
| 氏名 | 本人入力 |
| 会社名 | 本人入力（任意） |
| 役職 | 倫理法人会内の役職（その他は自由入力） |
| 所属委員会 | 5委員会＋その他（その他は自由入力） |
| 委員会役割 | リーダー／サブリーダー（該当時） |

- 同じ LINEユーザーID で再送信すると、その行を**上書き更新**します（役職・委員会の変更に対応）。

## 注意

- 役職・出席等はセンシティブ情報（設計書 §5.2）。スプレッドシートの共有範囲は役職者に限定し、共有パスワードを使わないこと。
- 仕様変更で列を増やすときは `Code.gs` の `HEADERS` と `row` を合わせて修正。
